SyncRoom 插件端 UI 原型

左侧是基于当前 popup 代码的忠实原型，右侧是一版可落地的简约高级感改版。 两版都保留房间、共享视频、LiveKit 语音、成员状态、重试提示和高级信息。

设计目标：不改变现有功能优先级，不弱化 B 站与通用 HTML5 视频需要展示的标题、来源、分享人和成员信息。 优化版只调整表现层，按钮反馈、语音波形和重试省略号都可以直接映射到 popup CSS。

04 / 动效与落地说明

视觉层级与按钮

当前房间作为主区域，使用轻雾蓝灰底，比普通卡片深一小档但不过度抢眼。未加入房间时展示创建/加入入口，房间码为空时“加入”、未入房时“同步当前视频”和麦克风都呈不可用态。复制邀请、退出房间、同步当前视频保持同一套中性按钮样式；hover 只做上浮、边框和阴影增强，不把背景变白。退出房间默认不显示红色，只有悬浮时才露出危险语义。

连接失败与重试

创建或加入房间时如果同步服务器不可达，按钮不能长期置灰，用户应能重试。 错误提示条保持窄而轻，进行中常驻显示“无法连接到同步服务器。重试（n/5）...”，省略号保持同一基线并错峰闪烁；最终失败后显示“重试 5 次后仍无法连接到同步服务器。”。

成员列表内语音控制

房间成员默认可语音，不再单独占用“小队语音”面板；语音连接状态合并到在线成员标题。当前用户右侧是可点击麦克风按钮，用来静音/开麦；其他成员右侧只展示麦克风状态。当前用户行只用很浅的灰色区别其他成员，hover 成员行时头像只旋转一次，离开时不反向旋转。

实现注意

高级设置展开态需要覆盖服务器地址、身份、重连、时钟同步和日志预览。优化版只改表现层，不改变现有数据结构；后续支持 B 站 + 通用 HTML5 视频时，应通过站点适配器扩展，不改动既有 B 站逻辑。插件端和 admin 都要继续保留视频标题、来源/BV 或 URL 摘要、分享人、用户名和语音状态，避免泛化后字段置空。
